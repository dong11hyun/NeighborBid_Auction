## 현재 상태 25_11_26
👤 회원 시스템 (Users)

회원가입 및 로그인: 관리자 페이지가 아닌 일반 사용자용 UI 제공.
신용도(평판) 모델: 사용자 정보에 reputation_score 필드 포함.
접근 권한: 로그인 여부에 따른 메뉴(마이페이지, 경매등록 등) 분기 처리.

💰 재화 및 지갑 (Wallet)

지갑 생성: 회원가입 시 자동 생성.
충전: 마이페이지에서 버튼을 눌러 재화 충전.
안전 결제 로직:
    입찰 시: 내 지갑 잔액 차감 → locked_balance로 이동 (돈 묶기).
    상위 입찰 발생 시: 기존 입찰자의 묶인 돈 즉시 환불(잠금 해제).
    낙찰/즉시구매 시: locked_balance 차감 → 판매자 balance로 입금 (정산).
거래 로그: 모든 돈의 흐름(충전, 입찰, 환불, 정산)을 Transaction 모델에 기록.

🔨 경매 시스템 (Auction)

상품 등록: 이미지 업로드, 시작가, 시간 설정 등 상세 정보 입력.
목록 및 검색: 키워드 검색, 필터링(가격순/최신순/마감임박순).
상세 페이지: 상품 정보 조회 및 판매자 정보 확인.
입찰하기: 최소 입찰 단위 검증 및 동시성 제어(DB Lock)가 적용된 입찰 기능.
즉시 구매: 설정된 가격 지불 시 즉시 경매 종료 및 정산.
수동 종료: 판매자가 종료 시간이 지난 경매를 직접 "낙찰 확정" 처리.

🤝 커뮤니티 및 편의 (Community)

문의/댓글: 상품 상세 페이지 하단 Q&A 게시판.
관심 상품(찜하기): 하트 버튼을 눌러 마이페이지에서 모아보기.
마이페이지: 내 지갑 잔액, 내가 입찰한 목록, 내가 올린 물건 관리

① 알림 시스템 (Notification) - [코드 적용 필요]
사용자가 매번 새로고침하지 않아도 중요한 사건을 알려주는 기능입니다.
 알림 기능: 데이터베이스에 알림을 저장하고 상단 뱃지로 표시.
 입찰 성공/실패 알림: "낙찰되었습니다!" 또는 "패찰했습니다." 알림.
 상위 입찰 발생 알림: "더 높은 입찰자가 나타나 밀려났습니다!" (환불 알림).
 내 경매 입찰 알림: 판매자에게 "누가 입찰했습니다!" 알림.
 정산 완료 알림: 판매자에게 "판매 수익이 입금되었습니다." 알림.

② 자동화 시스템 (Automation) - [난이도 상]
현재는 판매자가 버튼을 눌러야 끝나지만, 시스템이 알아서 해야 하는 기능들입니다.
 경매 자동 종료: 종료 시간이 되면 서버(Celery 등)가 감지하여 상태 변경.
 낙찰자 자동 결정: 종료 즉시 1등을 찾아 정산 프로세스 실행.
 자동 입찰 설정: 사용자가 상한선을 걸어두면 시스템이 대신 입찰 싸움을 해주는 기능.

③ 실시간성 (Real-time) - [난이도 최상]
 실시간 입찰 현황: 새로고침 없이 숫자가 50,000 → 55,000으로 촤르륵 바뀌는 기능 (WebSocket 필요).
 실시간 알림: 페이지 이동 없이 우측 상단에 팝업(Toast)이 뜨는 기능.

④ 정책 및 관리 (Policy & Admin)
 경매 개설 자격: 신용도가 낮거나 재화가 없으면 경매 등록 차단.
 보증금 예치: 경매 올릴 때 보증금을 걸고, 취소 시 차감하는 정책.
 어뷰징 탐지: 이상한 패턴의 입찰이나 자전거래 감시.
