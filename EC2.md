좋은 질문입니다. **Lightsail**은 사실 빠진 것이 아니라 **'시니어 개발자 관점'**에서 볼 때 EC2가 훨씬 더 강력한 어필 포인트가 되기 때문에 상대적으로 덜 강조한 것입니다.

이력서와 포트폴리오를 올리는 목적이 **"나 이만큼 편하게 구축했어"**가 아니라 **"나 이만큼 AWS 생태계를 잘 이해하고 있어"**라면 왜 **EC2**가 정답인지, 그리고 Lightsail과의 결정적 차이가 무엇인지 명확히 비교해 드릴게요.

---

### 1. 왜 EC2를 추천하는가? (시니어 개발자의 시선)

현업의 시니어 개발자나 면접관은 단순히 '서버가 돌아가는 결과'만 보지 않고 **'어떤 환경에서 어떻게 구축했는지'**를 더 중요하게 봅니다.

**① "진짜" 클라우드 환경 경험 (VPC, 서브넷, 보안그룹)**

* **Lightsail:** 클릭 몇 번이면 네트워크, 보안, 로드밸런서가 자동으로 세팅됩니다. 너무 편해서 내부가 어떻게 동작하는지 몰라도 됩니다.
* **EC2:** 깡통 컴퓨터를 받는 것과 같습니다. 외부에서 접속하려면 **Security Group(보안 그룹)**을 직접 열어야 하고, **VPC(가상 네트워크)** 개념을 알아야 하며, **IP 할당(Elastic IP)**도 직접 관리해야 합니다.
* **결론:** EC2를 썼다는 것은 *"아, 이 지원자는 포트 열고, 방화벽 설정하고, 네트워크 구조를 잡을 줄 아는구나"*라는 무언의 증명이 됩니다.

**② 확장성과 유연함 (AWS 생태계와의 연동)**

* 나중에 프로젝트가 커져서 DB를 분리하고 싶을 때(RDS), 이미지 파일을 따로 저장하고 싶을 때(S3), 도메인을 더 멋지게 관리하고 싶을 때(Route53) 등 AWS의 다른 서비스와 붙이기에 EC2가 훨씬 자연스럽고 표준적입니다.
* 현업의 90% 이상은 EC2 기반의 아키텍처를 사용합니다. 즉, EC2 경험은 **실무 투입 가능성**과 직결됩니다.

**③ 커스터마이징의 깊이**

* Lightsail은 정해진 OS와 정해진 스펙(패키지) 내에서 움직여야 하지만, EC2는 인스턴스 타입(CPU 최적화, 메모리 최적화 등)부터 OS의 세부 설정까지 완전히 내 마음대로 조작할 수 있습니다. 이는 백엔드 엔지니어에게 필수적인 덕목입니다.

---

### 2. Lightsail은 왜 추천 순위에서 밀렸나?

Lightsail이 나쁜 서비스는 절대 아닙니다. 오히려 **가성비**와 **편의성** 면에서는 압도적입니다. 하지만 '개발자 포트폴리오' 관점에서는 다음과 같은 이유로 추천 순위가 낮아집니다.

**① "너무 쉬워서" 어필이 약함**

* Lightsail은 워드프레스 블로그나 간단한 웹사이트를 빠르게 띄우기 위해 만들어진 **PaaS(Platform as a Service)**에 가깝습니다.
* 면접관 입장에서 Lightsail로 배포했다고 하면 *"그냥 편한 길을 선택했네"*라고 생각할 수 있지만, EC2로 밑바닥부터 세팅했다고 하면 *"인프라에 대한 도전 의식이 있네"*라고 평가합니다.

**② 확장성의 한계 (블랙박스)**

* Lightsail은 AWS가 많은 부분을 숨겨두고(추상화) 제공합니다. 트러블슈팅이 필요하거나 아주 디테일한 서버 튜닝이 필요할 때, 숨겨진 부분 때문에 답답할 수 있습니다.
* 물론 Lightsail에서 EC2로 업그레이드하는 기능이 있지만, 처음부터 EC2로 시작하는 것이 학습 곡선에는 더 도움이 됩니다.

### 3. 한눈에 보는 비교 (개발자 취업용)

| 비교 항목 | **AWS EC2** (추천) | **AWS Lightsail** |
| --- | --- | --- |
| **주요 타겟** | 엔터프라이즈, 전문 개발자 | 개인 블로거, 소규모 쇼핑몰, 초보자 |
| **네트워크 관리** | 직접 설정 (VPC, Security Group 등) | 자동/단순화 (방화벽 설정 제한적) |
| **확장성** | 무한함 (Auto Scaling, Load Balancer 등) | 제한적 (스케일업 위주) |
| **기술적 어필** | **"클라우드 인프라 이해도 높음"** | **"서버 띄울 줄 앎"** |
| **비용 (소규모)** | 프리티어(1년 무료) 활용 가능 | 월 $3.5~5 (매우 저렴하고 고정적) |
| **난이도** | 상 (리눅스 및 네트워크 지식 필요) | 하 (클릭 몇 번으로 끝) |
