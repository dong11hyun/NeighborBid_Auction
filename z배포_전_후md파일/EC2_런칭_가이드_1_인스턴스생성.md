## 1. EC2란 무엇인가요? (vs Lightsail)

이전에는 **Lightsail**이라는 "간편 세트 메뉴"를 추천해 드렸습니다.  
이제 여러분은 **EC2**라는 "커스텀 주문"의 세계로 오셨습니다.

| 특징 | Lightsail (세트 메뉴) | EC2 (커스텀 주문) |
| :--- | :--- | :--- |
| **비유** | 햄버거 세트 (버거+감튀+콜라) | 서브웨이 (빵, 치즈, 야채 하나하나 선택) |
| **장점** | 버튼 몇 번이면 끝. 가격이 고정됨. | 내 맘대로 성능 조절 가능. 확장성 무한대. |
| **단점** | 섬세한 설정 불가능. | 설정할 게 많음 (IP, 방화벽, 용량 등). |

하지만 걱정 마세요! 가장 최적의 조합(꿀조합)을 아래에서 그대로 알려드릴 테니 따라만 오세요. 🚀

## Step 1: 인스턴스(서버) 시작하기

### 1-1. AWS 콘솔 접속
1.  [AWS 콘솔](https://console.aws.amazon.com/)에 로그인합니다.
2.  오른쪽 상단에서 **"서울(Seoul)"** 리전을 확인합니다 (`ap-northeast-2`).
3.  상단 검색창에 **"EC2"**를 검색하고 클릭합니다.

### 1-2. 인스턴스 시작 버튼 클릭
1.  EC2 대시보드에서 주황색 **[인스턴스 시작]** 버튼을 클릭합니다.

### 1-3. 이름 및 OS 선택
화면을 아래로 내리면서 순서대로 설정합니다.

-   **이름(Name)**: `NeighborBid-Production` (원하는 이름 입력)
-   **애플리케이션 및 OS 이미지**:
    -   **[Ubuntu]** 아이콘 클릭
    -   버전 선택: **Ubuntu Server 22.04 LTS (HVM)**  
        *(최신 버전인 24.04도 있지만, 안정성을 위해 22.04를 추천합니다)*
    -   아키텍처: **64비트(x86)** (기본값 유지)

---

## Step 2: 인스턴스 유형 및 키 페어

### 2-1. 인스턴스 유형 (서버 사양)
어떤 성능의 컴퓨터를 빌릴지 정하는 단계입니다.

-   **추천 (무료 체험)**: `t2.micro` (RAM 1GB) - *프리 티어 사용 가능*이라고 써있습니다.
-   **추천 (실제 서비스용)**: `t3.small` (RAM 2GB) - *월 약 1.5만원~2만원*
    -   NeighborBid 프로젝트는 Redis, Channels 등을 쓰기 때문에 **2GB 램(`t3.small`)**을 강력 추천합니다. `t2.micro`는 램 부족으로 서버가 멈출 수 있습니다.

### 2-2. 키 페어 (로그인 열쇠)
서버의 비밀번호 대신 사용할 열쇠 파일(`.pem`)을 만듭니다. 이 파일은 **절대로 잃어버리면 안 됩니다!**

1.  **[새 키 페어 생성]** 클릭
2.  **키 페어 이름**: `NeighborBid-Key` (영어만 가능)
3.  **키 페어 유형**: `RSA` (기본값)
4.  **프라이빗 키 파일 형식**:
    -   Windows (구형 Putty 사용자): `.ppk`
    -   **Windows (PowerShell), Mac, Linux**: `.pem` **(추천! 우리는 이걸 쓸 겁니다)**
5.  **[키 페어 생성]** 버튼 클릭
    -   🚨 `NeighborBid-Key.pem` 파일이 내 컴퓨터에 다운로드됩니다.
    -   이 파일을 안전한 곳(예: `C:\Users\내이름\.ssh\`)에 잘 보관하세요.

---

## Step 3: 네트워크 및 보안 설정

이 부분이 EC2에서 가장 중요하고, 초보자가 많이 실수하는 부분입니다.
**"방화벽(보안 그룹)"**을 설정해서 누가 들어올 수 있는지 정해야 합니다.

1.  **네트워크 설정** 섹션에서 **[편집]** 버튼 클릭 (우측 상단)
2.  **보안 그룹 생성** 선택
3.  **보안 그룹 이름**: `NeighborBid-SG`
4.  **설명**: `Security group for NeighborBid Web Server`
5.  **인바운드 보안 그룹 규칙** (아래 규칙들을 하나씩 추가하세요):

| 유형 | 프로토콜 | 포트 범위 | 소스(Source) | 설명 |
| :--- | :--- | :--- | :--- | :--- |
| **SSH** | TCP | 22 | **내 IP** (또는 위치 무관 0.0.0.0/0) | 관리자 접속용 (키 파일 가진 사람만) |
| **HTTP** | TCP | 80 | **위치 무관 (0.0.0.0/0)** | 일반 사용자 웹 접속 |
| **HTTPS**| TCP | 443 | **위치 무관 (0.0.0.0/0)** | 보안 웹 접속 (필수!) |

> [!WARNING]
> 만약 DB(5432)나 Redis(6379) 포트를 여기서 열면 안 됩니다! 해킹 위험이 있습니다.
> 우리는 Docker 내부 네트워크를 쓸 거라 22, 80, 443만 열면 충분합니다.

---

## Step 4: 스토리지(하드디스크) 설정

-   **스토리지 구성**: 기본값(8GB)은 조금 부족할 수 있습니다.
-   **30GiB** 로 변경하는 것을 추천합니다. (프리 티어에서도 30GB까지 무료입니다)
-   볼륨 유형: `gp3` (최신) 또는 `gp2` 둘 다 괜찮습니다.

### 🚀 최종 시작
설정이 다 끝났습니다.
오른쪽 요약 화면을 보고 **[인스턴스 시작]** 주황색 버튼을 클릭하세요!

## Step 5: 고정 IP (Elastic IP) 할당

EC2는 껐다 켜면 IP 주소가 바뀝니다. 도메인을 연결하려면 **변하지 않는 IP(고정 IP)**가 필요합니다.
AWS에서는 이걸 **"탄력적 IP (Elastic IP)"**라고 부릅니다.

### 5-1. 탄력적 IP 생성
1.  왼쪽 메뉴에서 **네트워크 및 보안** -> **탄력적 IP** 클릭
2.  우측 상단 **[탄력적 IP 주소 할당]** 클릭
3.  설정 건드릴 것 없이 바로 하단 **[할당]** 클릭
    -   새로운 IP 주소(예: `3.12.34.56`)가 생겼습니다!

### 5-2. 인스턴스에 연결
방금 만든 IP를 우리가 만든 서버에 붙여야 합니다.

1.  생성된 IP 주소를 체크(✅)하고, 우측 상단 **[작업]** -> **[탄력적 IP 주소 연결]** 클릭
2.  **리소스 유형**: `인스턴스`
3.  **인스턴스**: 입력창을 클릭하면 아까 만든 `NeighborBid-Production` 서버가 뜹니다. 선택하세요.
4.  **프라이빗 IP 주소**: 자동으로 뜨는 IP 선택
5.  **[연결]** 클릭

### 🎉 1단계 완료!
이제 여러분의 서버는 **고정된 IP 주소**를 가지게 되었습니다.
이 IP 주소(예: `3.12.34.56`)를 메모장에 적어두세요. 앞으로 계속 쓰입니다.
