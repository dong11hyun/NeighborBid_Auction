# AWS IAM 동료에게 EC2 전체 권한 부여하기

> **목표**: 기존에 Lightsail 권한만 있던 동료ㄴ에게 **EC2를 자유롭게 다룰 수 있는 모든 권한**을 추가로 부여합니다.

---

## 1. AWS IAM 콘솔 접속

1.  [AWS 콘솔](https://console.aws.amazon.com/)에 **루트 계정**(또는 관리자 권한)으로 로그인합니다.
2.  상단 검색창에 **"IAM"**이라고 입력하고 클릭합니다.

## 2. 사용자(User) 찾기

1.  IAM 대시보드 왼쪽 메뉴에서 **[사용자]** (Users)를 클릭합니다.
2.  사용자 목록에서 동료의 이름인 ' '를 찾아 클릭합니다.

## 3. 권한 추가하기

이제 `DH` 님에게 새로운 권한 카드(정책)를 쥐여줄 차례입니다.

1.  **[권한]** (Permissions) 탭이 선택되어 있는지 확인합니다.
2.  오른쪽의 **[권한 추가]** 버튼을 누르고 -> **[권한 추가]** (Add permissions)를 선택합니다.
3.  **"권한 옵션"** 선택 화면에서:
    -   **[직접 정책 연결]** (Attach policies directly)을 선택합니다. (제일 오른쪽 버튼)

## 4. EC2 전체 권한 검색 및 선택

1.  아래쪽 **"권한 정책"** (Permission policies) 검색창에 다음을 입력합니다:
    -   검색어: `AmazonEC2FullAccess`
2.  검색 결과 리스트에서 `AmazonEC2FullAccess` 라는 이름 왼쪽에 있는 **체크박스(✅)**를 체크합니다.
    -   *설명: 이 권한은 EC2 인스턴스를 만들고, 끄고, 삭제하는 등 EC2에 대한 모든 행위를 허용합니다.*

## 5. (선택 사항) 탄력적 IP 및 기타 필수 권한

보통 EC2를 다루다 보면 IP 주소 관리(VPC)나 로드밸런서 권한도 필요할 때가 많습니다. `AmazonEC2FullAccess`에 대부분 포함되어 있지만, 확실하게 하기 위해 하나 더 추가하면 좋습니다. (필수는 아님)

-   같은 화면에서 검색어 지우고 `IAMUserChangePassword` 같은 게 있다면 동료가 스스로 비번을 바꿀 수 있게 해 줄 수도 있습니다.
-   일단은 **EC2 Full Access**만으로도 충분합니다.

## 6. 최종 저장

1.  맨 아래 **[다음]** (Next) 버튼 클릭.
2.  **검토** 화면에서:
    -   사용자: ` `
    -   추가된 정책: `AmazonEC2FullAccess`
    -   이게 맞는지 확인합니다.
3.  **[권한 추가]** (Add permissions) 버튼 클릭.

## 🎉 완료!

이제 **` `** 님은 AWS에 로그인하면 EC2 대시보드에서 모든 작업(인스턴스 생성, 중지, 설정 변경 등)을 수행할 수 있습니다.

> [!TIP]
> 권한은 즉시 적용되지만, 혹시 동료가 이미 로그인해 있다면 **"로그아웃 후 다시 로그인"** 해보라고 말씀해 주세요.
